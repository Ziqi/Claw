## 请示 1 回复: 界面选型 → 混合架构

**导师裁决：后端 API + CLI (高级用户) + Web UI (团队协作/态势感知)**

- TUI/CLI：适合安全分析师快速单点问答、日志查询
- Web Dashboard：多步推理场景不可替代（思维链可视化、工具调用记录、资产图谱）
- 缺乏可视化的智能体在企业环境被视为"黑盒"，难以获得信任

## 请示 2 回复: 模型选型 → LLM Routing (动态路由)

**导师裁决：动态模型切换是成熟架构实践**

| 模型层级 | 适用场景 | 示例 |
|---|---|---|
| Flash (轻量) | 高频低复杂度、日志提取、格式化 | IP 提取、XML→JSON、告警分类 |
| Pro (重型) | 战术决策、攻击链推演、报告撰写 | Kill Chain 推理、事件响应报告 |

## 请示 3 回复: 自主权边界 → HITL 三级分权

**导师裁决：状态变更操作必须强制人类审批**

| 等级 | 操作类型 | 审批策略 |
|---|---|---|
| 低风险 | 查询数据库、读文件、拉情报 | 自动执行 |
| 中风险 | 资产盘点、合规扫描 | 受范围和频率限制 |
| 高风险 | 封禁策略、主机隔离、凭据重置 | **强制 HITL 审批** |

> "智能体只能提供'建议执行的操作'及上下文证据，最终确认按钮必须由人类分析师按下。"
