## 请示 1: Agent 交互界面选型

将 CLAW 升级为智能体后，AI 的交互界面有三种可能：

| 方案 | 优势 | 劣势 |
|---|---|---|
| **A: 纯 TUI** | 轻量、可 SSH、与现有架构无缝 | 无法可视化攻击拓扑 |
| **B: Web Dashboard** | 可展示思维链 + 网络图谱 + 交互性强 | 需起本地 Web Server |
| **C: TUI + Web 混合** | Agent 循环走 TUI，可视化走 Web | 开发量中等 |

> 恳请导师定调：Agent 核心循环应采用哪种交互形态？

## 请示 2: 模型选型

Agentic 架构对模型推理能力要求极高。两个选项：

| 模型 | 推理深度 | API 成本 | 响应速度 |
|---|---|---|---|
| Gemini 3 Flash | 中 | 低 (适合高频扫描) | 快 |
| Gemini 3 Pro | 高 | 较高 | 较慢 |

> 是否采用**动态模型切换**策略：查库/简单任务用 Flash，复杂战术决策用 Pro？

## 请示 3: Agent 自主权边界 (安全策略)

Agent 在真实靶场中的最大自主权应到哪一步？建议分级：

| 等级 | 操作类型 | 审批 |
|---|---|---|
| L1 只读 | 查库、读文件、分析日志 | 自动 |
| L2 侦察 | nmap、httpx、curl | 自动 |
| L3 审计 | Nuclei、Nikto | 需确认 |
| L4 攻击 | psexec、Hydra、漏洞利用 | **强制人工审批** |
| L5 后渗透 | secretsdump、C2 投递 | **双重确认** |

> 导师认可此分级方案吗？L2（侦察类）是否可以开放自动执行？
